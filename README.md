# CULT_BOT 🤖

Консольный Python-бот для отправки сообщений в Telegram через Bot API.

---

## 🚀 Возможности

* 📤 Отправка сообщений в Telegram
* 🔐 Безопасное хранение токена через `.env`
* 🌐 Поддержка прокси
* ⚡ Простая настройка и запуск
* ⚠️ Базовые проверки конфигурации перед отправкой

---

## 📦 Структура проекта

```text
CULT_BOT/
├── bot.py
├── requirements.txt
├── .env.example
├── .gitignore
└── README.md
```

---

## ⚙️ Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Enigma333988/CULT_BOT.git
cd CULT_BOT
```

### 2. Установить зависимости

```bash
python -m pip install -r requirements.txt
```

---

## 🔑 Настройка

Создай файл `.env` на основе шаблона:

```bash
cp .env.example .env
```

Заполни:

```env
TOKEN=your_bot_token_here
CHAT_ID=-1001234567890
PROXY=
```

---

## 🧠 Переменные окружения

| Переменная | Обязательна | Описание |
| ---------- | ----------- | -------- |
| `TOKEN`    | Да          | Токен Telegram-бота |
| `CHAT_ID`  | Да          | ID личного чата, группы или канала |
| `PROXY`    | Нет         | Прокси в формате `http://...`, `https://...`, `socks5://...` или `socks5h://...` |

---

```bash
python bot.py
```

## 🔐 Безопасность

* Никогда не загружай `.env` в GitHub.
* Не публикуй `TOKEN` в чатах, скриншотах и логах.
* Используй `.env.example` только как шаблон без реальных секретов.
* Если токен уже утёк, немедленно перевыпусти его через **@BotFather**.
* По возможности используй отдельного бота для тестов и отдельного — для продакшена.
* Убедись, что `PROXY` указывает на доверенный прокси-сервер.

### Что уже сделано в коде

* `.env` исключён из Git через `.gitignore`.
* Переменные окружения проверяются при старте.
* Прокси подключается только если явно задан.
* Разрешены только ожидаемые схемы прокси (`http`, `https`, `socks5`, `socks5h`).
* Отключено наследование прокси/сетевых настроек из системных переменных окружения (`session.trust_env = False`).
* Ответ Telegram API обрабатывается более аккуратно: бот показывает статус/описание ошибки без лишнего шума.

## 🧠 Как это работает

1. Программа читает `.env`.
2. Берёт `TOKEN` и `CHAT_ID`.
3. Опционально добавляет прокси.
4. Отправляет HTTP-запрос в Telegram Bot API.
5. Показывает понятный результат отправки в консоли.

---

## 🌐 Telegram API

Используется официальный метод:

```text
https://api.telegram.org/bot<TOKEN>/sendMessage
```

---

## 📌 Примечания

* Если бот не отправляет сообщения — сначала проверь `TOKEN` и `CHAT_ID`.
* Если ошибка сети — проверь `PROXY` и доступ к `api.telegram.org`.
* Убедись, что бот добавлен в нужный чат и имеет необходимые права.
* Для каналов и супергрупп `CHAT_ID` часто имеет вид `-100...`.
