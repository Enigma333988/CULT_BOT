# CULT_BOT 🤖

Telegram-бот для отслеживания заказов мебели: администратор создаёт заказ, получает персональную ссылку и отправляет её покупателю, а покупатель открывает бота и сразу видит статус своего заказа.

---

## 🚀 Возможности

* 📦 Создание заказов прямо в Telegram командой `/neworder`
* 🔗 Персональная deep-link ссылка для каждого заказа
* 💳 Ввод полной цены и фактически полученной суммы с выводом в виде `50% (21.000 ₽ из 42.000 ₽)`
* 🧭 Гибкие статусы заказа, включая этапы доставки
* 📝 Примечания к заказу
* 🗂 Удобный список текущих заказов, отсортированный по дате создания
* 👤 Кнопка `Мои заказы` для клиента после первого открытия персональной ссылки
* 🔔 Мгновенные уведомления клиенту при смене статуса, оплаты, доставки, завершении и удалении заказа
* 🗃 Архив заказов в отдельных текстовых файлах и отчёт по периоду через `/report`
* ✅ Завершение заказа с подтверждением `Да / Нет`
* 🗑 Удаление заказа из бота
* ❤️ Автоматическое сообщение клиенту с благодарностью после завершения заказа
* 🌐 Публичный режим для пользователей без токена: контакты и соцсети Культ Мебель
* 🔐 Настройки и секреты через `.env`
* 🌐 Поддержка прокси

## 🔑 Настройка

Создай файл `.env`:

```bash
cp .env.example .env
```

Пример:

```env
TOKEN=your_bot_token_here
ADMIN_CHAT_ID=123456789
PROXY=
TIMEZONE=Europe/Moscow
ORDERS_FILE=orders.json
STATE_FILE=bot_state.json
ARCHIVE_DIR=archive
```

---

## 🧠 Переменные окружения

| Переменная | Обязательна | Описание |
| ---------- | ----------- | -------- |
| `TOKEN` | Да | Токен Telegram-бота |
| `ADMIN_CHAT_ID` | Да | Chat ID администратора, который создаёт и ведёт заказы |
| `PROXY` | Нет | Прокси в формате `http://...`, `https://...`, `socks5://...` или `socks5h://...` |
| `TIMEZONE` | Нет | Часовой пояс в формате IANA, например `Europe/Moscow` или `UTC` |
| `ORDERS_FILE` | Нет | Путь к JSON-файлу с заказами |
| `STATE_FILE` | Нет | Путь к JSON-файлу с сервисным состоянием polling |
| `ARCHIVE_DIR` | Нет | Папка, где хранятся текстовые архивы заказов |

> У бота должен быть установлен username через **@BotFather**, иначе deep-link ссылка для покупателя не будет работать.

## ▶️ Как пользоваться ботом

### 1. Запуск

```bash
python bot.py
```

### 2. Команды администратора

* `/start` — открыть меню администратора
* `/neworder` — создать новый заказ
* `/orders` — показать список текущих заказов
* `/report` — сформировать отчёт по периоду
* `/cancel` — отменить текущее создание заказа

### 3. Создание заказа

После `/neworder` бот по шагам спросит:

1. наименование товара;
2. полную цену — можно просто ввести `42000`, бот покажет `42.000 ₽`;
3. сколько уже фактически получено — например `21000`;
4. нужна ли доставка;
5. примечание к заказу.

Новый заказ создаётся автоматически со статусом `В ожидании`, после чего администратор может сразу менять статус кнопками в карточке заказа.

После этого бот создаст заказ и покажет персональную ссылку вида:

```text
https://t.me/<bot_username>?start=order_<token>
```

Эту ссылку нужно отправить покупателю.

### 4. Что видит покупатель

Если покупатель открывает персональную ссылку, бот автоматически показывает:

* наименование заказа;
* цену;
* процент оплаты;
* статус;
* есть ли доставка;
* примечание;
* напоминание о том, что срок изготовления указан в оферте.

Также покупателю доступны кнопки:

* `Мои заказы`
* `Связаться`
* `Соц.сети Культ Мебель`
* `Обновить статус`

Если покупатель уже однажды открыл персональную ссылку, бот запоминает его заказ и потом показывает кнопку `Мои заказы` даже при обычном `/start` без токена.

Если пользователь пишет боту без персонального токена и у него ещё нет привязанных заказов, бот объясняет, что это за бот, и даёт ссылки для связи и заказа.

### 5. Статусы заказа

Базовые статусы:

* `В ожидании`
* `Принято в работу`
* `Изготовление`
* `Покраска`
* `Сборка`
* `Заказ готов`

Дополнительные статусы для заказов с доставкой:

* `Ожидание доставки`
* `В пути`

### 6. Управление заказом

В карточке заказа администратору доступны кнопки:

* смена статуса;
* `Завершить заказ` — с подтверждением `Да / Нет`;
* `Удалить заказ`;
* возврат к списку заказов.

После завершения заказа клиенту отправляется сообщение с благодарностью.

Если заказ удалить, клиент также получает отдельное уведомление с кнопкой связи.

### 7. Отчёт по периоду

* `В ожидании`
* `Принято в работу`
* `Изготовление`
* `Покраска`
* `Сборка`
* `Заказ готов`

```text
/report
```

Далее бот ждёт период в формате:

```text
22 января 2025_1 сентября 2025
```

Отчёт строится по архивным текстовым файлам из `ARCHIVE_DIR`.

## 🧠 Как это работает

1. Администратор создаёт заказ в личном чате с ботом.
2. Бот сохраняет заказ в `orders.json`.
3. Для заказа создаётся уникальный токен и deep-link ссылка.
4. Покупатель открывает ссылку и видит только свой заказ.
5. Администратор в любой момент меняет статус, завершает или удаляет заказ.
6. После завершения заказа бот отправляет клиенту благодарность.

---

## 🔐 Безопасность и ограничения

* Не загружай `.env`, `orders.json` и `bot_state.json` в Git.
* Если токен утёк — перевыпусти его через **@BotFather**.
* Для корректной работы deep-link у бота должен быть username.
* `ORDERS_FILE` и `STATE_FILE` должны указывать именно на файл, а не на папку.
* Покупатель видит заказ только по персональной ссылке.
* Если заказ удалён, старая ссылка перестанет работать.
