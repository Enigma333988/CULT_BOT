# CULT_BOT 🤖

Telegram-бот для отложенных постов: принимает фото и текст прямо в Telegram, ставит публикации в очередь и автоматически публикует их по расписанию.

---

## 🚀 Возможности

* 📷 Приём фото прямо в Telegram
* 📝 Добавление текста к посту
* 📅 Планирование даты и времени публикации
* 🗂 Просмотр текущей очереди командой `/queue`
* 🚀 Автоматическая публикация в нужный чат/канал
* 🔐 Секреты и настройки через `.env`
* 🌐 Поддержка прокси

## 🔑 Настройка

Создай файл `.env`:

```bash
cp .env.example .env
```

Пример:

```env
TOKEN=your_bot_token_here
CHAT_ID=-1001234567890
ADMIN_CHAT_ID=123456789
PROXY=
TIMEZONE=Europe/Moscow
QUEUE_FILE=queue.json
STATE_FILE=bot_state.json
```

---

## 🧠 Переменные окружения

| Переменная | Обязательна | Описание |
| ---------- | ----------- | -------- |
| `TOKEN` | Да | Токен Telegram-бота |
| `CHAT_ID` | Да | Чат/канал, куда бот будет публиковать посты |
| `ADMIN_CHAT_ID` | Да* | Chat ID администратора, который управляет очередью. Если не указан, используется `CHAT_ID` |
| `PROXY` | Нет | Прокси в формате `http://...`, `https://...`, `socks5://...` или `socks5h://...` |
| `TIMEZONE` | Нет | Часовой пояс в формате IANA, например `Europe/Moscow` или `UTC` |
| `QUEUE_FILE` | Нет | Путь к JSON-файлу с очередью постов |
| `STATE_FILE` | Нет | Путь к JSON-файлу с сервисным состоянием polling |

> `ADMIN_CHAT_ID` лучше указать явно. Обычно это ваш личный chat id, а `CHAT_ID` — id канала или группы для публикации.

## ▶️ Как пользоваться ботом

### 1. Запуск

```bash
python bot.py
```

### 2. Открой диалог с ботом в Telegram

Отправь команду:

```text
/start
```

### 3. Создай новый отложенный пост

Команда:

```text
/newpost
```

Дальше бот пошагово попросит:

1. отправить фото;
2. отправить текст поста;
3. указать дату и время публикации в формате `YYYY-MM-DD HH:MM`.

Пример:

```text
2026-03-20 18:30
```

Бот сохранит пост в очередь и подтвердит время публикации.

### 4. Посмотреть очередь

Команда:

```text
/queue
```

Бот покажет все отложенные посты со временем публикации и кратким превью текста.

### 5. Отменить текущее создание поста

Команда:

```text
/cancel
```

## 🧠 Как это работает

1. Бот получает входящие сообщения через `getUpdates` (long polling).
2. Только `ADMIN_CHAT_ID` может управлять очередью.
3. Фото, текст и время публикации собираются пошагово.
4. Пост сохраняется в `queue.json`.
5. Когда наступает нужное время, бот публикует фото с подписью в `CHAT_ID`.
6. После публикации статус обновляется в очереди.

## 🔐 Безопасность и ограничения

* Не загружай `.env`, `queue.json` и `bot_state.json` в Git.
* Если токен утёк — перевыпусти его через **@BotFather**.
* Убедись, что бот добавлен в канал/группу и имеет право отправлять сообщения.
* Для каналов бот должен быть администратором.
* В очереди сейчас поддерживается формат **фото + подпись**.
* Лимит подписи Telegram для фото — до `1024` символов.
* Время вводится в часовом поясе из `TIMEZONE`.

## 📌 Полезные замечания

* Если хочешь узнать свой `ADMIN_CHAT_ID`, можно временно добавить вывод `message["chat"]["id"]` в лог или воспользоваться любым ботом/утилитой для просмотра chat id.
* Для канала `CHAT_ID` часто начинается с `-100...`.
* После перезапуска очередь не теряется, потому что хранится в `queue.json`.
