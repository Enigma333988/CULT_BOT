# CULT_BOT 🤖

Бот для отслеживания заказов мебели в **Telegram** и **MAX**: менеджер может создать заказ и вести его из любого подключённого мессенджера, а клиент — открыть персональную ссылку в удобном канале и видеть актуальный статус.

---

## 🚀 Что умеет бот

* 📦 Создавать заказы из каталога в Telegram **или** MAX.
* 🔄 Синхронизировать один и тот же заказ между двумя мессенджерами.
* 🏷 Помечать, где заказ был **создан** и в каких мессенджерах клиент его **открывал**.
* ✏️ Менять статус, оплату, доставку, завершать или удалять заказ из любого подключённого админского чата.
* 🔗 Генерировать персональные deep-link ссылки сразу для Telegram и MAX.
* 👤 Привязывать клиента к заказу отдельно по каждому мессенджеру.
* 🔔 Отправлять клиенту уведомления об обновлениях статуса во все подключённые им каналы.
* 🗂 Хранить каталог товаров, список заказов и архив по периодам.
* 📊 Показывать отчёт по архиву, в том числе с разбивкой по каналу создания заказа.

## 🔑 Настройка

Создай файл `.env`:

```bash
cp .env.example .env
```

Пример:

```env
TOKEN=your_telegram_bot_token_here
ADMIN_CHAT_ID=123456789
MAX_TOKEN=your_max_bot_token_here
MAX_ADMIN_CHAT_ID=123456789
PROXY=
TIMEZONE=Europe/Moscow
ORDERS_FILE=orders.json
STATE_FILE=bot_state.json
ARCHIVE_DIR=archive
CATALOG_FILE=catalog.json
MINI_APP_TITLE=CULT Mini App
MINI_APP_BUTTON_TEXT=Открыть приложение
MINI_APP_HOST=127.0.0.1
MINI_APP_PORT=8080
MINI_APP_PUBLIC_URL=
```

### Переменные окружения

| Переменная | Обязательна | Описание |
| ---------- | ----------- | -------- |
| `TOKEN` | Нет* | Токен Telegram-бота |
| `ADMIN_CHAT_ID` | Да, если есть `TOKEN` | Chat ID администратора в Telegram |
| `MAX_TOKEN` | Нет* | Токен MAX-бота |
| `MAX_ADMIN_CHAT_ID` | Да, если есть `MAX_TOKEN` | ID администратора в MAX |
| `PROXY` | Нет | Прокси в формате `http://...`, `https://...`, `socks5://...` или `socks5h://...` |
| `TIMEZONE` | Нет | Часовой пояс IANA, например `Europe/Moscow` или `UTC` |
| `ORDERS_FILE` | Нет | Путь к JSON-файлу с заказами |
| `STATE_FILE` | Нет | Путь к JSON-файлу со служебным состоянием polling |
| `ARCHIVE_DIR` | Нет | Папка, где хранятся текстовые архивы заказов |
| `CATALOG_FILE` | Нет | Путь к JSON-файлу каталога товаров |
| `MINI_APP_TITLE` | Нет | Заголовок стартовой страницы Mini App |
| `MINI_APP_BUTTON_TEXT` | Нет | Текст кнопки открытия Mini App в Telegram |
| `MINI_APP_HOST` | Нет | Локальный host для HTTP-сервера Mini App |
| `MINI_APP_PORT` | Нет | Локальный port для HTTP-сервера Mini App |
| `MINI_APP_PUBLIC_URL` | Нет | Публичный `https://` URL, который Telegram откроет как Mini App |

\* Нужно указать хотя бы один токен: `TOKEN` или `MAX_TOKEN`.

> Для deep-link в Telegram у бота должен быть username через **@BotFather**. Для MAX у бота также должен быть настроен публичный username.

```bash
python bot.py
```

Если настроены оба токена, процесс будет одновременно слушать Telegram и MAX.

### Mini App

При том же запуске бот поднимает Mini App на `http://127.0.0.1:8080/` или на значениях из `MINI_APP_HOST` и `MINI_APP_PORT`.

Чтобы Telegram открывал эту страницу прямо внутри приложения, нужно указать `MINI_APP_PUBLIC_URL` с публичным `https://` адресом. После этого бот зарегистрирует кнопку меню и команду `/miniapp` для запуска Mini App.

Теперь Mini App умеет:

* определять среду `browser / telegram / max / vk`;
* читать токен заказа из `?orderToken=...`, `?token=...`, `#orderToken=...` и `startapp=order_<token>`;
* запрашивать актуальный статус заказа через `GET /api/order?token=<token>`;
* показывать карточку заказа, прогресс по этапам, оплату и историю изменений.

Для VK Mini App можно использовать тот же URL, если он опубликован по `https://`. Если статический файл [miniapp/index.html](/C:/Users/shiro/Desktop/CULT_BOT/miniapp/index.html) лежит отдельно от Python-сервера, передайте адрес backend через `?apiBase=https://example.com`.

> Сейчас интеграция с MAX работает через **long polling**. Для production документация MAX рекомендует использовать **Webhook**.

## 🧭 Как это работает

### 1. Администратор

Админ может в Telegram и/или MAX:

* открыть меню `/start`
* создать заказ `/neworder`
* посмотреть активные заказы `/orders`
* вести каталог `/catalog`
* получить отчёт `/report`
* отменить текущее действие `/cancel`

Все изменения сразу записываются в единый `orders.json`, поэтому заказ, созданный в одном мессенджере, сразу виден в другом.

### 2. Создание заказа

После `/neworder` бот по шагам спросит:

1. товар из каталога;
2. сколько уже оплачено — `0%`, `50%`, `100%`;
3. нужна ли доставка;
4. примечание.

После создания бот показывает карточку заказа и ссылки для клиента:

```text
Telegram: https://t.me/<bot_username>?start=order_<token>
MAX: https://max.ru/<bot_username>?start=order_<token>
```

### 3. Клиент

Клиент может открыть заказ по персональной ссылке **в Telegram, в MAX или в обоих мессенджерах**. Бот запомнит, где именно был открыт заказ, и будет показывать это менеджеру в карточке заказа.

Клиент видит:

* название заказа;
* цену;
* сумму и процент оплаты;
* текущий статус с прогрессом;
* способ получения;
* примечание;
* дату доставки, если она назначена.

### 4. Синхронизация

У каждого заказа теперь хранятся:

* `created_via` — в каком мессенджере заказ создали;
* `customer_bindings` — в каких мессенджерах клиент открыл заказ.

Это позволяет:

* создавать заказ в Telegram и менять его статус в MAX;
* создавать заказ в MAX и дальше сопровождать его из Telegram;
* видеть у менеджера, через какой канал пришёл заказ и где клиент его читает;
* отправлять уведомления сразу во все каналы, которые клиент уже подключил.

### 5. Отчёт

Команда `/report` ждёт период в формате:

```text
22 января 2025_1 сентября 2025
```

Отчёт строится по архиву и показывает:

* количество заказов;
* активные / завершённые / удалённые;
* сумму заказов и оплат;
* остаток к оплате;
* сколько заказов было создано через Telegram и через MAX.

## 📝 Примечания по MAX

* Для отправки API-запросов используется заголовок `Authorization: <MAX_TOKEN>`.
* Deep-link MAX имеет вид `https://max.ru/<bot_username>?start=order_<token>`.
* Бот обрабатывает обновления `message_created`, `message_callback` и `bot_started`.

## 🧪 Минимальная проверка после настройки

1. Запусти `python bot.py`.
2. Открой `/start` в Telegram-боте и/или MAX-боте под админским аккаунтом.
3. Создай тестовый заказ.
4. Открой Telegram-ссылку заказа как клиент.
5. Открой MAX-ссылку того же заказа как клиент.
6. Измени статус в одном мессенджере и проверь, что обновление видно в другом.
